# Software Engineering Concepts

## How to Think About ANY Project

### Step 1: What does the user DO?
- List every action a user can take
- This becomes your feature list

### Step 2: What needs to be SAVED?
- If user closes app and comes back, what should still be there?
- This becomes your database design

### Step 3: Sketch the UI
- Draw it on paper first
- No code yet

### Step 4: Frontend vs Backend
- Just displaying stuff = Frontend
- Saving/fetching/secrets = Backend

### Step 5: Design your API
- What requests does frontend need to make?
- GET = read, POST = create, PUT = update, DELETE = remove

### Step 6: Pick your tools
- Think "what do I need?" then Google it

### Step 7: Build piece by piece
- One feature at a time
- Test each piece before moving on
